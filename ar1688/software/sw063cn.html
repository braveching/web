0.63改动:
  1. 增加OEM_10BASET编译选项, 用于仅使用KSZ8842的端口1, 并且强制它工作在10BaseT下.
  2. 在RoIP模块中交换COR/PTT信号的方式由RFC 2833改为SIP MESSAGE文本.
  3. 为RoIP双方都增加30秒的定时器, 限制某一方的最长连续发送时间.
  4. 修改直接IP呼叫情况下的一个DNS问题.
  5. SDCC\bin下工具的编译器从VS2008/VC9升级到VS2013/VC12, 需要安装x86版本的Visual C++ Redistributable Packages for Visual Studio 2013.
  6. 增加OEM_JOSN编译选项, 给需要保留用RFC 2833交换RoIP COR/PTT信号的厂商.
  7. 增加使用KSZ8842的RoIP模块硬件型号AR168RT.
  8. 利用PHY连接状态, 去掉了KSZ8842初始化过程中2秒的等待.
  9. 在version.h中增加SYS_MINI_RUN, 只有当它定义了的时候才启用RTL8019AS上RTP优先的Minu Run.

已知问题：
  1. 法语版本显示不对, 除中文英文外的其它语种也可能显示有问题.
  2. SDCC\bin\Manager.exe在Windows 7下运行时不能正确显示GB2312编码的中文调试信息.
